# .md 파일로 작성해보기

블로그를 작성하면서 알게 된 점이 있어 글을 작성해 본다.

글을 작성할 때마다 불편하게 커밋을 열 번 넘게 수정하며 해보니 너무 불편하다고 생각이 들어 여러 방법들을 모색해 보았다. 

VS Code에서 플러그인 형식으로 이용할 수도 있고, 로컬로 Jekyll을 설치해서 로컬 서버에 띄워 리액트의 핫 로드 방식처럼 편집할 수도 있다(예전에 이렇게 했던 기억이 어렴풋이 났다).

또 방법 중에 하나인 노션으로 작성하기.

즉 노션으로 글을 써서 .md 파일로 내보내기를 하여 헤더 부분만 추가해서 작업 경로에 넣으면! 역시 블로그에 바로 업로드가 가능하단 점이었다.

전에 실패했을 때에도  .md 형식으로 작성했었던 기억이 있는데, 이번에 사용하게 된 테마가 ‘_post’ 디렉토리에 .html로 되어있길래 형식을 똑같이 지켜야 하는 줄 알았다.

그러나 커밋 테스트를 해본 후, 그리고 찾아봤을 때 애초 Jekyll 구성이 markdown이든 html이든 모두 html 형식으로 표시를 해주기 때문에 두 확장자 사용해도 무방하다는 점을 알았다. 

편리하게 쓸 수 있는 방법을 알게 되었으니 친숙하게 작성할 수 있을 것 같다!
