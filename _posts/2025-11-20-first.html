Github Pages로 블로그 만들기!

이왕 블로그를 생성할 거면 깃헙 페이지로 한 번 시작해 보기로 했다. 사실은 1년 전에 깃헙 페이지 배포를 해보려고 마음먹고 시작했었으나 (Ruby를 설치하고··· 커스텀 테마를 적용해보고···)Jekyll 테마 적용을 하다가 삽질의 삽질 끝에 치명적인 에러 하나 때문에 배포를 못하고 결국 포기 후 방치를 했었다.

그러나 4학년 막학기 취업 준비를 위해 도움이 되는 수업을 듣게 되면서 기술 블로그를 작성해 보는 과제가 등장했고, 다시 해보기로 마음을 먹게 되었다. 다른 tistory, 네이버 블로그, velog 등 다양한 블로그 사이트들이 있지만 이왕 귀찮은 거 제일 품이 많이 든다는 github pages로 해보기로 결정했다.

1. 우선 레포지토리를 생성한다. 2. 레포지토리 이름이 곧 url에 명시 되기 때문에 이상한 이름은 피하는 게 좋다. 3. 레포지토리 settings에 들어가서 pages -> deploy 세팅을 마치면 간단하게 사이트가 배포가 된다!

당연히 처음 생성되는 페이지는 순백의 여백 화면이 등장하게 된다. 하지만 우리는 여기서 좀 더 이쁘장하게 만들어 볼 것이다. Jekyll 테마를 적용하면 방금 막 만든 것 같은 홈페이지도 있어 보이게 만들 수 있다^^!

Jekyll 테마 적용하기

Jekyll 테마 사이트https://jekyllthemes.io/ 에 접속해서 마음에 드는 테마 사이트를 고른다. 내가 고른 건 'Clean Blog' 이었다. https://jekyllthemes.io/theme/startbootstrap-clean-blog-jekyll

github 레포지토리로 제공하는 테마는 zip 형태로 다운받아서 압축을 풀어서 그대로 작업 경로에 덮어서 붙여쓰기를 하여 간단하게 '_config.yml' 파일을 수정해서 사용했다. 아무튼 기본적인 깃헙 페이지의 구조는 index.html로 시작하는 거니 천천히 페이지부터 구성해보는게 어떨까 싶어 시작해보기로 했다.
